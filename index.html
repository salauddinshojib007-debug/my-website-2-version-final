M4 Element Visibility Trigger CSS class: .form-success-message
After successful submit, a success <div class="form-success-message"> becomes visible. In GTM: create an Element Visibility trigger → CSS Selector .form-success-message → enable Observe DOM Changes. No page reload needed.

A custom JavaScript listener watches for a specific DOM event (formSuccess) dispatched by the form handler. The Custom HTML tag in GTM contains the listener; a Custom Event trigger catches customFormSuccess from dataLayer. Test: the event fires only on valid submit.

M6 dataLayer.push (Developer Method) event: formSubmission
The most robust method. The developer adds a window.dataLayer.push() call in the form callback. In GTM: create a Custom Event trigger for formSubmission. Add Data Layer Variables for formType and formPosition, then use them as GA4 event parameters.
✓ dataLayer.push sent! Check GTM Preview → formSubmission event → Variables.
⚠ Please fill in Name and Email.
M7 DOM Scraping (Last Resort) #dom-success-msg / .form-dom-success
After submit a message with id dom-success-msg and class form-dom-success appears. In GTM: create a DOM Element variable → CSS Selector .form-dom-success → attribute empty (gets innerText). Then a DOM Ready trigger fires when that variable equals "Thanks for subscribing!".
